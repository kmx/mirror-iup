General

  • Build a MacOS X version using the new GTK Darwin native framework. This implicates in a CD using GDK/Cairo or using Quartz 2D.
  • The actual model for control data structure in the internal SDK is limited and hard to use for derived classes.
  • Move the SDK headers to the "include/sdk" folder.
  • Simplify IupLua implementation. More similar to a regular binding implementation like CDLua and IMLua.
  • Drag&Drop between controls or dialogs in the same application.
  • A MacOS X native driver using Carbon.
  • A tutorial section in the documentation. One big Controls Demo just like GTK, wxWidgets and Qt have.
  • Unicode Support (UTF-8) in Windows and Motif.
  • Define and control keyboard navigation in "Tab" groups.
  • Add support for loadind and saving RTF files in IupText. Add support for images inside the text.
  • Support for text over image in IupLabel, IupButton and IupToggle. Or a way to composte text on IupImage.
  • Support for changing the system menu in Windows. Support for cascading IupPopup for menus.
  • The IupImgLib takes an very long time to compile under Visual C++. In a 2.5GHz CPU tooks about 5min!

Windows

  • In Windows Vista the COMPOSITE=YES attribute of the IupDialog is not working as expected. There is still some flicker when the dialog is resized. So we changed the default to NO in Vista.
  • IupTabs using native controls in Windows Vista when COMPOSITE=YES works only if MULTILINE=YES, so it is internally overwritten until the problem is solved. (since 3.0)
  • The MDI Window menu is not properly updated when the child mdi is closed.

New Controls

  • DropDownButton - Mix between a drop down list and a button
  • ScrollBox -  a box that can scroll the elements inside
  • GridBox - container to distribute elements in a grid.
  • LayoutBuilder - a dialog that shows the layout of another dialog and allows the user to reposition the elements interactively.

  • ImageList - A list that contains images
